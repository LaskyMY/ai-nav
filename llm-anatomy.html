LEVEL 1 · 万亿参数级

模型架构 · 一切的开端

这是LLM最宏观的一层——决定模型"长什么样"。目前所有主流LLM（GPT-4、Claude、Gemini、DeepSeek）都基于同一个核心发明。

Transformer
2017 · Vaswani et al.
一切的基础。用Self-Attention替代RNN的循环结构，让模型能同时看到所有token的位置关系。训练并行化成为可能 → 参数规模从此爆炸。GPT中的"T"就是Transformer。
Decoder-only
GPT系列 · 自回归生成
只保留Transformer的Decoder部分。从左到右逐token预测下一个词。GPT-4、Claude、Llama、DeepSeek全是这个架构。Encoder-Decoder(T5、BART)则保留两部分用于翻译/摘要。ChatGPT的"G"是Generative——生成式。
MoE · 混合专家
Mixture of Experts
把一个大模型拆成多个"专家"子模型。每个token只激活部分专家（如DeepSeek-V3: 671B总参数 → 每token仅激活37B）。稀疏激活让模型能做到万亿参数但推理成本可控。GPT-4也被传使用MoE。
Attention
Scaled Dot-Product
整个Transformer的心脏。Q·K^T/√d → Softmax → ×V。让每个token能动态关注序列中所有其他token。公式虽简，复杂度O(n²)随长度爆炸——这就是上下文窗口消耗巨量算力的根源。
LEVEL 2 · 千卡GPU集群级

训练范式 · 它怎么学

有了架构，下一步是"教"模型。一个LLM不是一步训练完的——它经历多个阶段，每个阶段用不同的数据和目标。

Pre-training占训练算力的 99%+，消耗数万GPU·月的算力。SFT/RLHF/DPO 只需几天到几周。但后三个阶段决定了模型好不好用——是否对齐人类意图。

Pre-training
自监督学习 · Next Token
给模型喂万亿级token的文本，任务简单到极致：预测下一个token。就是"下一个字是什么？"反复问了万亿次。数据来自网页(C4/CommonCrawl)、代码(GitHub)、书籍、论文。输出：Base Model（基座模型）——还不会聊天，只会续写。
SFT · 指令微调
Supervised Fine-Tuning
用高质量对话示例(human-written)教模型问答格式。"用户问什么→AI怎么答"。数据量远小于预训练(几十万条对话)，但让模型从"续写器"变成"助手"。这就是为什么Base Model只会补全而ChatGPT会聊天。
RLHF
人类反馈强化学习
人对多个回答排序打分→训练一个奖励模型→用PPO强化学习微调LLM。让模型学会偏好人类喜欢的回答风格（有用、无害、诚实）。Claude的"Constitutional AI"是RLHF的变体——用AI替代人类做排序。
DPO
Direct Preference Optimization
2023年的突破——不需要先训练奖励模型，直接从偏好数据优化。把RLHF的两步(训奖励模型→PPO)合并为一步。更稳定、更快、更简单。Llama 3、Qwen等都用DPO做最终对齐。
LEVEL 3 · 数十层深度

网络组件 · 一层层拆开看

现在钻入一层Transformer block内部。每个Block = Attention + FFN + LayerNorm + Residual。GPT-4据说有120层这样的Block堆叠。

Multi-Head Attention
多头注意力
把Attention复制N份(GPT-3有96个头)，每头关注不同的语义关系。一个头可能关注语法结构，另一个关注指代关系，另一个关注语义相似性。多头的输出拼接后做一次线性变换→送入FFN。
FFN · 前馈网络
Feed-Forward Network
每个token经过Attention后，独立通过一个两层MLP：升维→激活→降维。例如升到4倍维度再压回来。这里存储了模型大部分的事实知识（研究表明FFN层是知识存储的主要位置）。
Layer Normalization
层归一化
对每层的输出做归一化——均值0方差1。没有它，深层网络的梯度会爆炸或消失。现代LLM用RMS Norm(只除均方根，不算均值)替代经典LayerNorm——更快，效果几乎一样。Llama和DeepSeek都用RMS Norm。
Residual Connection
残差连接 · 跳跃线
每层的输入直接加到输出上 → output = Layer(input) + input。让信号可以"跳过"任何一层直达后面。这使得120层的网络也能训练——梯度通过残差路径畅通无阻地回传。ResNet (2015)的遗产。
🔑一层的完整流程：输入 → (Multi-Head Attention → Add残差 → RMS Norm) → (FFN → Add残差 → RMS Norm) → 输出到下一层。这个pattern重复N次(N=层数)。
LEVEL 4 · 数十亿参数

参数矩阵 · 模型的"数字记忆"

每一层的Attention和FFN最终都存储为巨大的矩阵。这些矩阵中的浮点数是模型从数据中学到的"知识"。

1,760,000,000,000
GPT-4 估算参数量 (1.76万亿) · 每个参数是一个float16浮点数 · 约占 3.5TB 显存
Weights · 权重
模型的核心记忆
每个矩阵元素是一个浮点数（如0.042或-1.357）。QKV投影矩阵、FFN的上下投影矩阵——全是权重。训练就是不断调整这些数字以减小预测误差。GPT-4的1.76T参数=1.76万亿个这样的数字。
Embedding · 嵌入
Token → 向量
模型第一层：把离散的token ID映射到连续向量空间（如768维或4096维）。语义相近的词向量也相近。嵌入矩阵大小 = 词表大小 × 嵌入维度。GPT-3词表50257 × 12288维 ≈ 6亿参数仅用于嵌入。
LoRA · 低秩适配
Low-Rank Adaptation
微调时不碰原权重，在旁路加两个小矩阵(A×B，秩r=8~64)。参数量仅原模型的0.1%~1%却能达到接近全量微调的效果。让普通人在一张4090上就能微调Llama 70B。QLoRA进一步量化到4bit。
Quantization
量化 · 16bit→4bit
把参数从FP16(2字节)压缩到INT4(0.5字节)，模型体积缩小4倍。GPTQ、AWQ、GGUF(q4_K_M/q5_K_M)是主流方案。4bit量化后的70B模型可以在24GB显存的单卡上跑——这就是本地跑大模型的秘密。
LEVEL 5 · 逐token生成

推理过程 · 一个字一个字地想

训练好的模型如何实际"说话"？每生成一个新token，都要完整跑一遍整个网络。看似流畅的对话背后是上千次完整的前向传播。

Context Window
上下文窗口
模型能"一次看到"的最大token数。GPT-4 Turbo 128K、Claude 200K、Gemini 1M。窗口越大→Attention矩阵越巨大(O(n²))→显存消耗指数增长。这就是为什么长文本非常贵。超过窗口的历史对话会被截断。
KV Cache
键值缓存
每次生成新token时，之前所有token的Key和Value矩阵复用不重算。否则对话越长生成越慢。KV Cache是推理显存的主要消耗者——长对话下KV Cache可比模型权重还大。vLLM的PagedAttention专门优化这个。
Logits → Softmax
从分数到概率
最后一层输出logits向量(每个词表token一个分数，如词表50000维)。Softmax将这些分数转换为概率分布：P(token_i) = e^logit_i / Σe^logit。所有token的概率之和=1。这就是模型对"下一个字该是什么"的全部信念。
Sampling
温度 · Top-p · Top-k
Temperature: T→0 贪婪选最高概率(机械重复)；T→1 保留原始分布(更随机)。Top-p: 只从累积概率≥p的最小token集合中采样。p=0.9→过滤掉最离谱的10%。Top-k: 只保留概率最高的k个token。三者组合控制生成文本的确定性vs创造性。
LEVEL 6 · 最小原子

Token · 模型的"原子"

一切——万亿参数、120层深度、数万GPU——最终都是为了在数十亿token上做一件简单的事：预测下一个token。

Llama
"Llama" 这个单词被分词为 [L] [lam] [a] → 3个token。汉字通常1字=1~2个token
Tokenizer
分词器 · 文本→数字
把人类文本切成token序列并映射为整数ID。主流算法：BPE(Byte Pair Encoding，GPT系列用)、SentencePiece(Llama、T5用)。"Hello world!" → [15496, 995, 0]。没有tokenizer，模型不识字。
BPE · 字节对编码
Byte Pair Encoding
从字节级开始，反复合并最频繁的相邻字节对→形成子词(subword)。高频词保持完整(e.g. "the")，低频词被切碎(e.g. "antidisestablishmentarianism" → 多个子词)。平衡了词表大小和编码效率。OOV(未登录词)问题被彻底解决——所有Unicode都能用字节表示。
Vocabulary · 词表
模型的"字典"
所有可能token的集合。GPT-2词表50257、Llama 32000、DeepSeek-V3 129280。词表大小直接影响logits层的计算量——每一步都要为词表中每个token算一个分数。多语言模型需要更大词表。
Token ID
每个token的数字身份
最终，一个token就是一个整数。"the" = 278, "。" = 511, "GPT" = [383, 12, 344]。这个整数送入Embedding层→变成向量→开始在整个网络中流转。从万亿参数到最终输出，起点和终点都是这一个小小的整数。
💡完整链路回顾：你的输入→Tokenizer切成token IDs→Embedding变向量→120层Transformer(Attention+FFN+Norm+Residual重复)→最后一层输出logits→Softmax变概率→Sampling选一个token ID→Tokenizer解码回文字→显示在屏幕上。每个字的背后，模型都跑了一次完整的万亿参数前向传播。
