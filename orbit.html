神奇代码 · 核心就这几行

半隐式欧拉法 —— 最简稳定轨道模拟

整个轨道模拟的核心物理就 3 行代码。秘密在于：先用新速度更新位置，再计算加速度更新速度——顺序一换，稳定性天差地别。

// 1. 用当前速度更新位置（先算位置！）
pos.x += vel.x * dt;
pos.y += vel.y * dt;

// 2. 计算引力加速度
dx = sun.x - pos.x;
dy = sun.y - pos.y;
r = Math.sqrt(dx*dx + dy*dy + 1e-6);  // 软化因子防除零
ax = G * sun.mass * dx / (r*r*r);
ay = G * sun.mass * dy / (r*r*r);

// 3. 用新加速度更新速度（后算速度！）
vel.x += ax * dt;
vel.y += ay * dt;
🔑核心洞察：顺序是关键。显式欧拉先用旧加速度更新速度再更新位置 → 能量发散。半隐式欧拉反过来——先用速度更新位置，再算力，再更新速度 → 能量奇迹般稳定。就这一个交换，轨道从螺旋发散变成完美闭合。

Velocity Verlet —— 科研级辛积分

真正的"神奇代码"——每个时间步只计算一次力，却达到二次精度且能量守恒百万年不漂移。

// Velocity Verlet 三步走
pos.x += vel.x * dt + 0.5 * acc.x * dt * dt;  // 半步位置
pos.y += vel.y * dt + 0.5 * acc.y * dt * dt;

accNew = computeAcceleration(pos);  // 唯一次力计算

vel.x += 0.5 * (acc.x + accNew.x) * dt;  // 平均加速度更新速度
vel.y += 0.5 * (acc.y + accNew.y) * dt;
acc = accNew;

RK4 —— 经典四阶龙格-库塔

每步计算 4 次力，但精度和稳定性极高。适合需要高精度的短期模拟。

// k1 = 当前斜率
k1v = acc(pos) * dt;  k1p = vel * dt;
// k2 = 用 k1 推进半步后的斜率
k2v = acc(pos + 0.5*k1p) * dt;  k2p = (vel + 0.5*k1v) * dt;
// k3 = 用 k2 推进半步后的斜率
k3v = acc(pos + 0.5*k2p) * dt;  k3p = (vel + 0.5*k2v) * dt;
// k4 = 终点斜率
k4v = acc(pos + k3p) * dt;  k4p = (vel + k3v) * dt;
// 加权平均
vel += (k1v + 2*k2v + 2*k3v + k4v) / 6;
pos += (k1p + 2*k2p + 2*k3p + k4p) / 6;

开普勒定律 · 从公式到画面

三条定律决定一切轨道

第一定律 · 椭圆轨道
行星轨道是椭圆，太阳位于一个焦点。偏心率 e 决定"扁"的程度：e=0 完美圆，e→1 极扁椭圆。
第二定律 · 面积速度恒定
行星与太阳的连线在相等时间内扫过相等面积。这解释了为什么行星在近日点跑得快、远日点跑得慢——你可以在上面的模拟中亲眼看到。
第三定律 · 周期-半长轴关系
T² ∝ a³ —— 轨道周期的平方与半长轴的立方成正比。越远的行星，转一圈越慢。v = √(GM/r) 是圆形轨道的精确解。

为什么说它"颠覆"？

传统方法 vs 辛积分

传统轨道模拟用显式欧拉法（课本最常教的方法），但能量会不断增长——模拟几圈轨道就螺旋飞走了。而半隐式欧拉和 Velocity Verlet 这种辛方法 (Symplectic Integrator) 能保持哈密顿量（能量）在真实值附近振荡百万年不漂移。

最神奇的是：代码反而更简单。半隐式欧拉就比显式欧拉多了一个"先用速度更新位置"的步骤——代码行数一样，顺序一变，天壤之别。

💡启示：数值分析中，"更好的方法"往往不是"更复杂的方法"。理解问题的数学结构（这里是哈密顿力学的辛结构），反而能找到更简洁、更稳定、更优雅的解法。这也是为什么物理学家偏爱辛积分——不是因为它复杂，恰恰是因为它简单且正确。
