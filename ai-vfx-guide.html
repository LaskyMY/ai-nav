安装 Claude Code —— AI 编程核心载体

Claude Code 是什么？

Claude Code 是 Anthropic 推出的命令行 AI 编程助手。它不是 IDE 插件，而是一个直接运行在终端里的智能代理——可以读代码、写代码、执行命令、操作文件系统。我们把它作为整个工作流的"大脑"。

安装步骤

  1. 打开终端（macOS: Terminal.app / Windows: PowerShell 或 WSL2）
  2. 执行全局安装命令
  3. 验证安装成功
Terminal 
# 全局安装 Claude Code CLI
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version
# 输出示例: claude-code/2.x.x (darwin-arm64, node v22.x)
⚠️ 如果安装报权限错误（macOS/Linux），请在命令前加 sudo。如果网络不通（国内环境），可设置 npm 镜像：npm config set registry https://registry.npmmirror.com

首次启动 & 认证

  1. 在终端输入 claude 并回车
  2. 浏览器会自动打开 Anthropic 登录页
  3. 使用 Anthropic 账号登录（支持 Google / GitHub OAuth）
  4. 登录成功后终端会显示 "Authenticated"
  5. 如果使用第三方 API（后续步骤），可以跳过此认证
💡 这一步只是确保 Claude Code 本体安装正确。接下来我们会把它切换到 DeepSeek API，所以免费额度不够也完全没关系。
2

接入 DeepSeek V4 Pro API —— 高性能低成本模型

为什么要换模型？

DeepSeek V4 Pro（即 DeepSeek-V3.1-Terminus）提供了与 Claude 高度兼容的 API，且价格仅为 Claude 的 1/5 到 1/10。对于频繁的特效生成任务，成本差距非常可观。它原生支持 Anthropic Messages API 格式，所以 Claude Code 可以无感切换。

Step 2.1 — 获取 DeepSeek API Key

  1. 打开 https://platform.deepseek.com/api_keys 注册/登录
  2. 点击「创建 API Key」，命名为 claude-code-vfx
  3. 复制生成的 key（格式 sk-xxxxxxxxxxxxxxxx），妥善保存
  4. 建议先充值 ¥10-20 用作测试，API 调用按量计费

Step 2.2 — 配置 Claude Code Router（推荐方案）

Claude Code Router（CCR）是一个轻量级路由层，它拦截 Claude Code 的 API 请求，智能转发到 DeepSeek。好处是：

  • 可以在多个模型间切换（DeepSeek 编码 + Claude 审查）
  • 不需要修改 Claude Code 本身
  • 支持 Web UI 可视化管理
Terminal 
# 1. 安装 Claude Code Router
npm install -g @musistudio/claude-code-router

# 2. 启动 Web 配置界面
ccr ui
# 浏览器打开 http://localhost:XXXX 进行可视化配置

在 Web UI 中配置以下信息：

  • Provider Name：deepseek
  • API Base URL：https://api.deepseek.com/anthropic
  • API Key：填入你在 Step 2.1 获取的 sk-xxx
  • Model：deepseek-chat（常规）/ deepseek-reasoner（深度思考）

Step 2.2 替代方案 —— 直接环境变量（更简单）

如果你不需要多模型切换，直接把环境变量写入 Shell 配置文件是最快的方式：

~/.zshrc（或 ~/.bashrc） 
# DeepSeek Anthropic 兼容端点
export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
export ANTHROPIC_AUTH_TOKEN="sk-你的DeepSeek_API_Key"
export ANTHROPIC_MODEL="deepseek-chat"
export ANTHROPIC_SMALL_FAST_MODEL="deepseek-chat"

保存后执行 source ~/.zshrc 使其生效。之后启动 claude 就会自动走 DeepSeek API。

⚠️ DeepSeek 的 Tool Calling（工具调用）能力弱于 Claude 原生模型。对于简单的代码生成完全够用，但复杂的多步编辑可能偶尔失败需要重试。这也是为什么教程推荐 Router 方案——你可以配置 fallback 到 Claude 处理复杂任务。

Step 2.3 — 验证 API 连通性

Terminal 
# 测试 DeepSeek API 是否可达
curl -X POST https://api.deepseek.com/anthropic/v1/messages \
  -H "x-api-key: sk-你的Key" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{"model":"deepseek-chat","max_tokens":50,"messages":[{"role":"user","content":"Say hello"}]}'

# 如果返回包含 "content" 的 JSON，说明连接成功

配置 MCP Server —— 打通 AI 与游戏引擎

MCP（Model Context Protocol）是什么？

MCP 是 Anthropic 提出的开放协议，让 AI 模型可以安全地调用外部工具。对游戏特效来说，MCP Server 就是一座桥——AI 通过它直接操作 Unreal Engine 或 Unity Editor，比如创建 Niagara 粒子系统、修改材质、生成蓝图节点。

一句话：没有 MCP，AI 只能给你文字建议；有了 MCP，AI 直接在引擎里帮你做特效。

UE5 方案选择

MCP 方案工具数架构费用推荐场景
UnrealClaudeMCP 60 原生 C++ 插件 + Python 桥 免费 MIT 喜欢开源、想完全本地化
Rekall 480+ 纯 C++ 引擎内 FAB 付费 专业特效师、Niagara/Material 全覆盖
Flop MCP 广泛 HTTP 远程 免费/付费 不想装插件、远程协作

Unity 方案选择

MCP 方案工具数架构费用推荐场景
GladeKit MCP 222+ Python HTTP Bridge 免费 MIT 深度项目感知、语义搜索
Coplay MCP 86 TCP Bridge Beta 免费 社区最大、上手最快
GameCodex 5 (知识) stdio 免费 MIT 纯知识参考、无需引擎控制

推荐组合（新手最佳路径）

Unreal Engine 用户选 UnrealClaudeMCP（免费开源，60 个工具覆盖 Niagara/蓝图/材质/动画）。Unity 用户选 GladeKit MCP（222 个工具，社区活跃）。

引擎侧插件安装 —— UE5 / Unity 端配置

方案 A：Unreal Engine 5（推荐 UnrealClaudeMCP）

Step 4A.1 — 克隆 MCP 仓库

Terminal 
# 克隆 UnrealClaudeMCP
git clone https://github.com/NAJEMWEHBE/UnrealClaudeMCP.git
# 作用：获取 MCP Server 的完整代码，包含 C++ 插件和 Python 桥接

Step 4A.2 — 安装 C++ 插件到 UE 项目

Terminal 
# 将插件复制到你的 UE5 项目
cp -r UnrealClaudeMCP/Plugin/UnrealClaudeMCP "你的UE项目路径"/Plugins/

# 作用：让 UE 编辑器加载 C++ 插件，暴露编辑器 API 给外部调用

Step 4A.3 — 编译并启动

  1. 用 Visual Studio 打开你的 UE 项目，编译（Ctrl+Shift+B）
  2. 打开 UE 编辑器，确认 Plugin 已启用（Edit → Plugins → 搜索 "ClaudeMCP"）
  3. 回到终端，启动 Python 桥接
Terminal 
# 安装 Python 依赖
pip install -r UnrealClaudeMCP/bridge/requirements.txt

# 启动桥接（保持终端打开）
python UnrealClaudeMCP/bridge/unreal_claude_mcp_bridge.py
# 作用：建立 stdio-TCP 通信管道，一端连 Claude Code，一端连 UE 编辑器

Step 4A.4 — 注册 MCP 到 Claude Code

Terminal 
# 将 MCP Server 注册到 Claude Code
claude mcp add unreal -- python "/完整路径/UnrealClaudeMCP/bridge/unreal_claude_mcp_bridge.py"

# 作用：告诉 Claude Code "有一个叫 unreal 的工具集可用"
# 之后在 Claude Code 中输入自然语言指令，它就会自动调用 UE 工具

方案 B：Unity（推荐 GladeKit MCP）

Step 4B.1 — 安装 GladeKit MCP

Terminal 
# 通过 uvx 一键安装（uv 是 Python 的快速包管理器）
# 如果没有 uv，先安装: curl -LsSf https://astral.sh/uv/install.sh | sh

uvx gladekit-mcp
# 作用：自动拉取 GladeKit，启动 Unity 通信桥接

Step 4B.2 — 安装 Unity 侧插件

  1. 下载 GladeKit Unity Package（从 GitHub Releases）
  2. 在 Unity Editor 中：Assets → Import Package → Custom Package
  3. 导入后，Unity 顶部菜单栏会出现 "GladeKit" 菜单
  4. 点击 GladeKit → Start Server（默认端口 8765）

Step 4B.3 — 注册到 Claude Code

Terminal 
# 注册 GladeKit MCP
claude mcp add unity -- uvx gladekit-mcp

# 作用：Claude Code 获得 Unity Editor 的 222+ 个操作能力

方案 C：不需要引擎控制？用 GameCodex（1 分钟配置）

如果你只是想获得游戏特效开发的知识参考和代码建议，而不需要 AI 直接操作引擎：

Terminal 
# 一行命令获得 957+ 篇游戏开发文档
claude mcp add gamedev -- npx -y gamecodex
# 作用：为 Claude Code 注入 UE/Unity/Godot 等 29 个引擎的策展知识
# 适合：学习阶段、方案设计、代码审查

常见问题速查

Claude Code 提示 "MCP connection refused"

检查三点：① UE/Unity 编辑器是否完全启动 ② MCP 桥接脚本是否在运行 ③ 端口是否被防火墙阻拦（macOS 一般无此问题）

DeepSeek API 返回报错 / 长上下文出错

DeepSeek 建议长上下文阈值设为 60000 tokens。在 ~/.claude/settings.json 中添加："maxContextTokens": 60000

生成的 Niagara 效果和描述不符

正常现象。AI 目前对参数的理解是模糊的——把它当成"快速原型工具"：AI 搭架子（80%），你微调参数（20%）。效果好于从零手写。

UE5 插件编译失败

确认：① Unreal Engine 版本 ≥ 5.5 ② Visual Studio 安装了 C++ 游戏开发工作负载 ③ 项目是 C++ 项目而非纯蓝图项目
