安装 Claude Code —— AI 编程核心

Claude Code 是什么

Claude Code 是 Anthropic 推出的命令行 AI 编程代理。它不是 IDE 插件，而是一个运行在终端的完整 AI 助手——能读取文件、写入代码、执行 Shell 命令、调用外部工具（通过 MCP）。我们把它作为整个工作流的"操作中枢"。

安装命令

Terminal 
# 全局安装
npm install -g @anthropic-ai/claude-code

# 验证
claude --version
# 预期输出: claude-code/2.x.x
⚠ 权限不足时加 sudo。国内网络慢可设镜像：npm config set registry https://registry.npmmirror.com
💡 这一步安装的是 Claude Code 本体，下一步会把它切换到 DeepSeek API，所以不需要 Anthropic 付费账号。

接入 DeepSeek V4 Pro API —— 高性能 & 低成本

为什么要换模型

DeepSeek V4 Pro（deepseek-chat / deepseek-reasoner）提供了与 Anthropic Messages API 完全兼容的端点，价格仅 Claude 的 1/5~1/10。对大规模特效生成任务来说，成本差距非常显著。Claude Code 无需任何修改即可对接。

Step 2.1 · 获取 API Key

  1. 浏览器打开 platform.deepseek.com/api_keys
  2. 注册 / 登录 DeepSeek 开放平台
  3. 点击「创建 API Key」，命名为 claude-code-vfx
  4. 复制 key（格式 sk-xxxxxxxxxxxxxxxx）并保存
  5. 建议先充值 ¥10 测试用（按量计费，极便宜）

Step 2.2 · 配置环境变量（推荐方案）

将以下内容追加到 Shell 配置文件中。这是最简单、最可靠的方式。

Terminal · 选择你的 Shell 
# Bash 用户:
echo 'export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"' >> ~/.bashrc
echo 'export ANTHROPIC_AUTH_TOKEN="sk-你的Key"' >> ~/.bashrc
echo 'export ANTHROPIC_MODEL="deepseek-chat"' >> ~/.bashrc
echo 'export ANTHROPIC_SMALL_FAST_MODEL="deepseek-chat"' >> ~/.bashrc
source ~/.bashrc

# Zsh 用户 (macOS 默认):
echo 'export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"' >> ~/.zshrc
echo 'export ANTHROPIC_AUTH_TOKEN="sk-你的Key"' >> ~/.zshrc
echo 'export ANTHROPIC_MODEL="deepseek-chat"' >> ~/.zshrc
echo 'export ANTHROPIC_SMALL_FAST_MODEL="deepseek-chat"' >> ~/.zshrc
source ~/.zshrc

环境变量说明：ANTHROPIC_BASE_URL 指向 DeepSeek 兼容端点；ANTHROPIC_AUTH_TOKEN 是你的 API Key；ANTHROPIC_MODEL 指定模型。

Step 2.3 · 验证 API 连通性

Terminal 
# 测试 DeepSeek API
curl -s -X POST https://api.deepseek.com/anthropic/v1/messages \
  -H "x-api-key: sk-你的Key" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{"model":"deepseek-chat","max_tokens":50,"messages":[{"role":"user","content":"回复OK"}]}' | python3 -m json.tool

# 预期输出: 包含 "content" 字段的 JSON，内容为 "OK" 或类似回复
# 如果返回错误，检查 API Key 是否正确、是否有余额

备选方案 · Claude Code Router（多模型切换）

如果你需要同时使用 DeepSeek 和 Claude，或在它们之间智能切换：

Terminal

# 启动 Web 配置界面

配置 MCP Server —— 打通 AI 与游戏引擎

MCP（Model Context Protocol）是什么

MCP 是 Anthropic 制定的开放协议，让 AI 可以通过标准化接口调用外部工具。对游戏特效而言：

  • 没有 MCP：AI 只能给你文字建议，你得手动在引擎里操作
  • 有 MCP：AI 直接创建 Niagara 粒子系统、修改材质节点、生成蓝图逻辑、批量管理资产

MCP Server 就是一座桥——一端连着 Claude Code，一端连着游戏引擎编辑器的 API。

引擎侧安装 —— 插件部署 + 桥接启动

方案 A · Unreal Engine 5（UnrealClaudeMCP）

4A.1 · 克隆仓库

Terminal 
git clone https://github.com/NAJEMWEHBE/UnrealClaudeMCP.git
# 克隆到当前目录，获得 C++ 插件源码 + Python 桥接脚本

4A.2 · 安装 C++ 插件到 UE 项目

Terminal 
# 将插件复制到你的 UE5 项目
cp -r UnrealClaudeMCP/Plugin/UnrealClaudeMCP "/path/to/your/UE5Project"/Plugins/

# 示例:
# cp -r UnrealClaudeMCP/Plugin/UnrealClaudeMCP ~/Documents/UnrealProjects/MyGame/Plugins/

4A.3 · 编译 & 启用

  1. 用 Visual Studio 2022 打开项目的 .sln 文件
  2. 按 Ctrl+Shift+B 编译解决方案
  3. 打开 UE5 编辑器 → Edit → Plugins → 搜索 "ClaudeMCP" → 勾选启用
  4. 重启编辑器

4A.4 · 安装 Python 依赖 + 启动桥接

Terminal · 保持此窗口运行 
# 安装依赖
pip install -r UnrealClaudeMCP/bridge/requirements.txt

# 启动桥接（CTRL+C 停止）
python UnrealClaudeMCP/bridge/unreal_claude_mcp_bridge.py
# 预期输出: Bridge listening on 127.0.0.1:18888

4A.5 · 注册 MCP 到 Claude Code

Terminal 
# 替换路径为你的实际路径
claude mcp add unreal -- python "/绝对路径/UnrealClaudeMCP/bridge/unreal_claude_mcp_bridge.py"

方案 B · Unity（GladeKit MCP）

4B.1 · 安装 uv（Python 包管理器）+ GladeKit

Terminal 
# 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 安装并启动 GladeKit
uvx gladekit-mcp

4B.2 · 安装 Unity 插件

  1. 从 GitHub Releases 下载 GladeKit Unity Package
  2. Unity Editor → Assets → Import Package → Custom Package → 选择下载的 .unitypackage
  3. 导入后顶部菜单会出现 "GladeKit" → 点击 Start Server（默认 :8765）

4B.3 · 注册到 Claude Code

Terminal 
claude mcp add unity -- uvx gladekit-mcp

常见问题 & 排查

Q: Claude Code 启动后显示 "MCP connection refused"

检查顺序：① UE/Unity 编辑器是否完全启动 ② MCP 桥脚本是否在运行 ③ 端口是否被占用（lsof -i :18888 检查）

Q: DeepSeek API 报 400 错误 / 长上下文截断

DeepSeek 建议限制上下文为 60000 tokens。在 ~/.claude/settings.json 中设置：

{"maxContextTokens": 60000}

Q: 生成的 Niagara 效果和描述有偏差

正常。AI 对数值参数的理解是模糊的。把它当"快速原型工具"：AI 搭框架(80%)，你微调参数(20%)。比从零手写快 5-10 倍。

Q: UE 插件编译失败

三个可能：① UE 版本 < 5.5（需要 5.5+）② VS 没装 C++ 游戏开发工作负载 ③ 项目是纯蓝图项目（需要 C++ 项目）。解决方案：File → New C++ Class → 创建一个空类 → 关闭编辑器 → 编译 → 重新打开。

Q: 安装过程中 npm 报 permission denied

# macOS/Linux
sudo npm install -g @anthropic-ai/claude-code

# 或者使用 nvm 管理 Node 版本（推荐）
# curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Q: DeepSeek 工具调用不稳定，编辑经常失败？

DeepSeek 的 Tool Calling 能力确实弱于 Claude 原生。这是已知限制。建议方案：安装 Claude Code Router，配置 DeepSeek 处理常规对话 + Claude 处理复杂编辑任务，智能路由自动切换。
